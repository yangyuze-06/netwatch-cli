# netwatch-cli V0.9 Plan

## 目标

V0.9 聚焦 LibreSpeed custom server list，不开发 Playwright，不开发 HTTP file download test，不逆向 `speedtest.cn` 私有 API。

## 问题背景

Official Ookla CLI 在当前环境下可能选到 Hong Kong / Tokyo / Korea / Russia 等节点，结果明显低于 `speedtest.cn` 网页中的“广东移动_Vixtel_1”。这主要是测速服务器池不匹配，不应误判为用户本地宽带差。

LibreSpeed 是开源测速后端，支持从远程 JSON 或本地 JSON 加载服务器列表。它可以给用户一个更可控的 server list 入口，但公共节点质量不保证，也不能承诺复现 `speedtest.cn` 的专属/合作节点结果。

## 实现范围

- `netwatch/speedtest_backends/librespeed_cli.py`
  - 支持 `run_speedtest(server_json_url=..., local_json_path=..., duration=..., source_ip=...)`。
  - 支持 `--server-json <url>`。
  - 支持 `--local-json <file>`。
  - 支持 `--duration <seconds>`。
  - 支持 `--source <ip>`。
  - `server_json_url` 和 `local_json_path` 互斥。
  - 使用 `subprocess.run(list)`，不使用 `shell=True`。
  - 捕获 timeout、CalledProcessError、JSONDecodeError，返回 `SpeedtestResult.error`。

- `netwatch/config.py`
  - 新增 `preferred_librespeed` 配置。
  - 只保存 `mode`、`server_json_url`、`local_json_path`、`duration`。
  - 不保存 token、cookie、公网 IP。

- `netwatch/speedtest_runner.py`
  - 新增 `run_librespeed_custom_speedtest(...)`。
  - 直接参数和保存配置均可运行。
  - 不复用 Ookla `server_id` 语义。

- `netwatch/cli.py`
  - 高级功能新增“LibreSpeed 自定义服务器列表测速”。
  - 子菜单支持使用已保存配置、输入远程 URL、输入本地文件、保存配置、清除配置。
  - 明确提示公共节点质量不保证，结果取决于 server list 中的服务器质量。

## 配置结构

```json
{
  "preferred_librespeed": {
    "mode": "server-json",
    "server_json_url": "https://example.com/servers.json",
    "local_json_path": null,
    "duration": 15
  }
}
```

## 边界

- 不真实验证 URL 是否高质量。
- 不承诺 CLI 一定跑满千兆。
- 不承诺复现 `speedtest.cn` 的“广东移动_Vixtel_1”。
- 不逆向 `speedtest.cn` 私有 API。
- 自动测试必须 mock 公网测速、subprocess、requests。

## 后续方向

- 支持保存多个 LibreSpeed server list。
- 增加 LibreSpeed server list 展示和备注。
- 如果用户有自建 LibreSpeed 节点，可增加更明确的自建节点使用说明。
- HTTP file download test 作为独立方向后续评估，不属于 V0.9。
