# netwatch-cli V0.6 Plan

## 问题背景

V0.5 已经能从 ARP cache 补全 MAC，但 hostname 仍经常显示为 `-`。原因是很多设备不会暴露 hostname，路由器也未必提供反向 DNS。实际验证中，小米路由器后台可以显示设备名、IP 和 MAC，因此可以把路由器设备列表作为一个只读补充来源。

## 小米 LuCI API 调研

本机验证：

- `route -n get default` 可识别默认网关 `192.168.31.1`。
- `curl -I http://192.168.31.1/` 返回正常后台页面。
- 访问 `http://192.168.31.1/cgi-bin/luci/api/misystem/devicelist` 返回：

```json
{"code":401,"msg":"Invalid token"}
```

这说明设备列表 API 存在，但需要登录后的 `stok` token。常见接口：

```text
http://<router-ip>/cgi-bin/luci/;stok=<STOK>/api/misystem/devicelist?mlo=1
http://<router-ip>/cgi-bin/luci/;stok=<STOK>/api/misystem/devicelist
```

## 安全边界

- 不自动爆破。
- 不绕过登录。
- 不保存路由器密码。
- `stok` 默认只在本次运行内存中使用，不落盘。
- API 只读，不实现任何修改路由器配置的接口。
- API 失败时回退到 ARP / reverse DNS / mDNS 结果。

## 实现方案

- 新增 `netwatch/router.py`：
  - `get_default_gateway()`
  - `open_router_admin()`
  - `extract_xiaomi_stok()`
  - `fetch_xiaomi_device_list()`
  - `normalize_xiaomi_device()`
  - `merge_devices()`
- 主菜单新增：
  - 打开路由器后台。
  - 从小米路由器同步设备名。
  - 高级功能。
- 高级功能保留：
  - 列出 Speedtest 服务器。
  - 手动指定 Speedtest server id 测速。
  - 显示测速后端信息。
- 统一设备结构：
  - `NetworkDevice`
  - 表格显示 `Name | IP | MAC | Source | Status`
