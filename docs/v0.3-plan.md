# netwatch-cli V0.3 Plan

## 目标

V0.3 聚焦三个体验问题：Speedtest 单位更清楚、测速服务器可手动选择、局域网扫描结果更有用。

## Speedtest 单位

`speedtest-cli` 返回 download/upload 的单位是 bit/s。V0.3 同时显示：

- `Mbps = bit/s / 1_000_000`
- `MB/s = bit/s / 8 / 1_000_000`

输出示例：

```text
Download: 500.00 Mbps / 62.50 MB/s
Upload: 100.00 Mbps / 12.50 MB/s
```

## Speedtest 服务器选择

自动选择测速服务器可能因为服务器负载、运营商路由、跨网质量而波动。V0.3 增加：

- 运行 Speedtest 自动测速。
- 选择服务器测速。
- 列出附近测速服务器。
- 用户输入 server id 后针对该服务器测速。
- 自动测速失败或结果异常时，提示尝试手动选择服务器。

测速流程保持阶段提示：

- `正在选择最佳服务器...`
- `正在测试下载速度...`
- `正在测试上传速度...`

失败时输出错误原因，不显示 traceback。

## 局域网扫描增强

扫描结果表格改为：

```text
IP | MAC | Hostname | Status
```

主机名获取顺序：

1. `socket.gethostbyaddr(ip)`
2. `arp -a` 中的 hostname
3. 查不到则显示 `-`

macOS 下优先从 `arp -a` 解析 IP 和 MAC。厂商信息暂不实现，后续可以基于 MAC OUI 数据库补充 vendor 字段。

## 主机名缺失原因

部分设备无法显示主机名通常是正常现象：

- 设备未开放 hostname。
- 路由器不提供反向 DNS。
- iOS/macOS 隐私地址会减少可识别信息。
- 防火墙阻止发现。
