# netwatch-cli V0.7 Plan

## 背景

V0.6 已经支持扫描局域网、打开路由器后台、从小米 LuCI API 同步设备名。但主菜单功能偏碎，“从小米路由器同步设备名”更适合放在局域网设备发现流程中。实际 API 返回还可能出现同 IP 的重复记录，例如一条完整设备记录和一条空记录。

## 交互重构

主菜单调整为：

```text
1. 实时网卡流量
2. 本机网络信息
3. 局域网设备发现
4. 带宽测速
5. 打开路由器管理后台
6. 高级功能
7. 退出
```

局域网设备发现子菜单：

```text
1. 快速扫描：Ping + ARP
2. 增强扫描：Ping + ARP + 路由器设备名
3. 仅从路由器同步设备列表
4. 返回主菜单
```

## 路由器管理后台

入口改为通用“路由器管理后台”，提供默认网关和常见品牌入口：

- 当前默认网关。
- `192.168.31.1` / `miwifi.com`
- `192.168.0.1`
- `192.168.1.1`
- `192.168.50.1`
- `tplinkwifi.net`
- `router.asus.com`
- 手动输入。

## 小米设备去重

新增 `deduplicate_router_devices()`：

- 优先按 MAC 合并。
- MAC 缺失时按 IP 合并。
- name 选择非空且不是 `-` 的值。
- ip/mac 选择非空值。
- online 任一记录在线即在线。
- connect_type 合并非空字段。

这样可以避免同一个 IP 同时显示“有名记录 + 空记录”。

## Speedtest 高级功能降级

服务器列表仍保留在高级功能中。如果 `speedtest-cli` 后端返回 403，会提示：

```text
当前 speedtest-cli 后端无法获取服务器列表。建议使用“带宽测速”自动模式，或后续安装官方 Ookla CLI。
```

## 测试

新增去重测试覆盖：

- 同 IP 有一条完整记录和一条空记录。
- 同 MAC 多条记录。
- 无 MAC 但同 IP 多条记录。
- name 优先保留有意义名称。
