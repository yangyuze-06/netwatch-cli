# netwatch-cli V0.8 Plan

## 问题背景

Python `speedtest-cli` 和 Homebrew `speedtest-cli` 都是社区版工具，实际测速结果可能明显低于网页测速或官方客户端。官方 Ookla CLI 是独立的 `speedtest` 命令，支持 JSON 输出、server id、服务器列表和 selection details，因此 V0.8 将测速能力重构为多后端架构。

## 调研结论

- 官方 Ookla CLI：推荐默认后端，命令为 `speedtest`。
- LibreSpeed CLI：开源备选后端，测速网络和 Ookla 不同。
- Python speedtest-cli：最后 fallback，保留兼容，但结果仅供参考。

macOS 官方 Ookla CLI 安装：

```bash
brew tap teamookla/speedtest
brew install speedtest
```

不要把 `brew install speedtest-cli` 当作官方 Ookla CLI。

## 后端架构

新增：

- `netwatch/speedtest_backends/models.py`
- `netwatch/speedtest_backends/ookla_cli.py`
- `netwatch/speedtest_backends/librespeed_cli.py`
- `netwatch/speedtest_backends/python_speedtest.py`
- `netwatch/speedtest_runner.py`

统一模型：`SpeedtestResult`。

优先级：

1. `official-ookla-cli`
2. `librespeed-cli`
3. `python-speedtest-cli`

## 代理出口测速设计

新增 `netwatch/proxy_probe.py`：

- 优先调用 `curl -s --max-time 8 https://ipinfo.io/json`。
- curl 不可用时用 `requests` fallback。
- 只在终端展示当前 CLI 进程看到的公网出口，不保存公网 IP。

菜单新增“代理/当前出口测速”，流程：

1. 显示出口 IP、城市、国家/地区、ISP/组织。
2. 提示 TUN/VPN 与浏览器代理的区别。
3. 调用最佳可用测速后端。

## 安全边界

- 不绕过网络限制。
- 不破解代理。
- 不保存公网 IP。
- 只检测当前 CLI 进程实际看到的公网出口。
